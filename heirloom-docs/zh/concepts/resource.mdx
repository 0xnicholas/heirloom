**资源（Resource）**是 Heirloom 的一等业务实体。客户、订单、合同、资产和策略都被建模为资源。

资源不是数据库行。它们拥有独立的生命周期、稳定的全局标识符和完整的事件历史。

## 资源结构

| 字段 | 含义 | 对智能体的意义 |
|-------|---------|---------------------------|
| `rid` | 全局、不可变的资源标识符 | 跨数据迁移保持稳定引用 |
| `type` | 资源类型名称 | 定义能力、字段和状态机 |
| `owner` | 当前所有者 | 驱动权限传播 |
| `state` | 当前生命周期状态 | 决定哪些操作合法 |
| `fields` | 业务属性 | 可读写的结构化数据 |
| `relationships` | 指向其他资源的链接 | 可遍历的依赖图 |
| `events` | 不可变事件流 | 上下文与审计的完整历史 |
| `version` | 单调递增版本号 | 乐观并发控制 |

## RID

资源标识符（`rid`）全局唯一且永不改变。它让智能体能以 `ri.customer.123` 引用同一个客户，无论底层数据今天存在于 PostgreSQL、明天在 BigQuery、后年通过 REST API 提供。

## 资源生命周期

1. 在 Schema Registry 中**定义**资源类型。
2. 从连接的数据源**物化**实例。
3. 通过受能力与状态约束的操作**交互**。
4. 通过状态机转移和事件触发器**自动化**。
5. 通过提案与分支**治理**模式变更。
6. 根据智能体使用模式**演化**类型系统。

## 资源类型生命周期

资源类型本身经历以下阶段：

```
DISCOVERED → ACTIVE → EVOLVING → STALE → DEPRECATED → DELETED
```

阶段转换通过提案工作流触发。
