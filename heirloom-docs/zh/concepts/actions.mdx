---
title: 操作
description: 操作是 Heirloom 中唯一的结构化写入路径。每个操作都通过九步验证流水线。
---

**操作（Action）**是改变资源状态的唯一方式。它是一个结构化写入操作，具有目标资源、所需能力、状态门、校验规则和副作用。

操作在本体中定义，由运行时强制执行。不满足所有门禁的请求会被拒绝并记录。

## 九步流水线

| 步骤 | 名称 | 职责 |
|------|------|----------------|
| 1 | **认证** | 解析调用者身份：人类、智能体或自动化 |
| 2 | **角色** | 查询调用者在相关范围的角色 |
| 3 | **能力** | 派生调用者当前有效的能力 |
| 4 | **门禁** | 检查能力是否覆盖所需能力 |
| 5 | **状态** | 验证目标当前状态是否允许该操作 |
| 6 | **校验** | 运行业务规则验证 |
| 7 | **执行** | 写入资源存储并更新索引 |
| 8 | **事件** | 追加不可变事件，成功或被拒 |
| 9 | **通知** | 向下游订阅者发布变更事件 |

## 操作定义规则

注册操作时必须满足三条规则：

1. **能力门禁** —— 所需能力必须在目标资源类型上声明。
2. **状态门禁** —— 门禁状态必须存在于目标状态机中。
3. **类型一致** —— 目标与参数类型必须匹配 Schema Registry。

违反任一规则，操作定义即被拒绝。不安全操作无法注册，因此永远不能被调用。

## 通知操作

**通知操作**是一种特殊操作子类型，没有目标资源。它用于发送消息或调用 webhook 等外部副作用。它跳过状态和校验步骤，但仍经过认证、角色、能力、事件和通知。

## 示例：审批订单

```json
{
  "name": "approve_order",
  "targetType": "Order",
  "requires": ["mutate"],
  "gate": {"state": "pending"},
  "validate": [
    {"function": "risk_score", "inputs": {"order_rid": "#target.rid"}}
  ]
}
```

调用 `approve_order` 的智能体必须持有包含 `Order.mutate` 的能力，且目标订单必须处于 `pending` 状态。
