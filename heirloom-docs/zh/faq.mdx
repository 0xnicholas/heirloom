## Heirloom 是什么？

Heirloom 是一个 AI 原生的语义本体系统。它在企业数据与 AI 智能体之间提供类型安全层，使智能体能够查询并作用于业务数据，而无需直接数据库访问。

## 与数据目录有何不同？

数据目录告诉智能体数据*在哪里*。Heirloom 告诉智能体数据*意味着什么*、允许什么操作——然后通过操作、能力和审计强制执行这些操作。

## 与图数据库有何不同？

图数据库让你遍历关系。Heirloom 增加了精确的关系语义（所有权、引用、关联）、类型级能力、状态机和统一治理链。

## 智能体能删除数据吗？

仅当资源类型显式声明 `drop` 能力时。如果未声明 `drop`，则任何角色——包括管理员——都无法创建删除操作。安全由类型系统强制执行，而非提示词。

## 智能体有独立的安全模型吗？

没有。智能体、人类和自动化都流经相同的 `行为体 → 角色 → 能力票据 → 操作` 链。唯一区别是它们持有的角色。

## Heirloom 使用什么查询语言？

Heirloom 使用为 LLM 友好的 JSON DSL。它支持过滤、排序、分页、聚合、关系遍历和混合语义搜索。Query Resolver 将其转换到底层源。

## Heirloom 存储我的数据吗？

Heirloom 主要是现有数据源之上的语义层。资源是 PostgreSQL、REST API、流和文件上的物化视图。知识库确实在 Heirloom 管理的存储中索引 Markdown 文章。

## SDK 支持哪些语言？

目前支持 Python（`heirloom-sdk`）。未来可能增加其他语言 SDK。

## 我可以在自己的基础设施上运行 Heirloom 吗？

可以。服务器是一个连接 PostgreSQL 的 Spring Boot 应用。你可以在本地、容器或任何云提供商上运行。

## 在哪里查看 API 文档？

服务器运行时，访问 http://localhost:8080/docs 查看交互式 Swagger/OpenAPI 文档。
