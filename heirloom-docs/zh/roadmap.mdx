Heirloom 的路线图遵循四阶段成熟度模型，从只读查询层到可信自主操作。

## 成熟度模型

| 阶段 | 重点 | 成果 |
|-------|-------|---------|
| **1. 语义查询层** | 跨源统一只读查询 | 智能体通过稳定语义界面查询业务数据 |
| **2. 安全操作层** | 带能力、角色、能力票据的操作层 | 智能体执行受治理的写入并留下完整审计 |
| **3. AI 智能体集成** | Agent SDK、函数、语义搜索、监控 | 智能体成为一等消费者 |
| **4. 治理与规模化** | 分支、条件能力、多后端 | 企业级本体治理 |

## 当前状态

截至最新路线图更新：

- **Phase 0: 核心基础设施** —— 完成。Schema Registry、REST/GraphQL API 和知识库管线已运行。
- **Phase 1: 语义查询层** —— 大体完成。JSON DSL、Mapping Engine、Query Resolver 和 Perspective Engine 已就位。
- **Phase 2: 安全操作层** —— 大体完成。能力、操作流水线、角色/能力模型、状态机和治理提案已实现。
- **Phase 3: AI 智能体集成** —— 大体完成。Python SDK、函数沙箱、语义搜索和审计面板已可用。
- **Phase 4: 治理与规模化** —— 部分完成。本体分支、知识版本化和跨本体映射已完成。条件能力和专用图/事件存储待实现。
- **Phase 5: 可信自治** —— 探索性。高级智能体协作、形式方法和生态市场是未来工作。

## 今天即可使用

- 定义带字段、能力和状态机的资源类型。
- 跨映射数据源运行语义 JSON DSL 查询。
- 注册知识源、同步 Markdown 文章并搜索。
- 定义带九步验证流水线的操作。
- 为人类用户和智能体分配角色与能力。
- 从智能体或操作内调用沙箱函数。
- 审计行为体活动、检测异常并重放决策链。

## 即将到来

- 基于 CDC 的增量同步。
- 专用图存储后端（如 Neo4j）。
- 与状态、时间或上下文绑定的条件能力。
- 最小能力集的agent角色模板。
- agent可达边界的正式验证。
